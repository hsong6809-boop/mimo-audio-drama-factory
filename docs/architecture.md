# 系统架构设计

## 整体架构

```
用户输入（题材/设定）
        │
        ▼
┌─────────────────────────────────────────────┐
│              Orchestrator 编排器              │
│  负责整个流水线的调度、状态管理、错误恢复     │
└─────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────┐
│            Phase 1: 文本生成层               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ 选题Agent │→│ 剧本Agent │→│ 质检Agent │  │
│  └──────────┘  └──────────┘  └──────────┘  │
└─────────────────────────────────────────────┘
        │ 输出：结构化剧本（JSON）
        ▼
┌─────────────────────────────────────────────┐
│            Phase 2: 声纹设计层               │
│  ┌──────────────┐  ┌──────────────────────┐ │
│  │ 角色属性提取  │→│ 音色匹配/创建        │ │
│  └──────────────┘  └──────────────────────┘ │
│  ┌──────────────┐  ┌──────────────────────┐ │
│  │ 情感标注     │→│ TTS参数映射          │ │
│  └──────────────┘  └──────────────────────┘ │
└─────────────────────────────────────────────┘
        │ 输出：带声纹信息的剧本
        ▼
┌─────────────────────────────────────────────┐
│            Phase 3: 语音合成层               │
│  ┌──────────────┐  ┌──────────────────────┐ │
│  │ 情感TTS引擎  │→│ 音频片段库           │ │
│  └──────────────┘  └──────────────────────┘ │
└─────────────────────────────────────────────┘
        │ 输出：按角色/段落分割的音频片段
        ▼
┌─────────────────────────────────────────────┐
│            Phase 4: 音频编排层               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ 片段拼接  │→│ BGM混音  │→│ 后期处理  │  │
│  └──────────┘  └──────────┘  └──────────┘  │
└─────────────────────────────────────────────┘
        │ 输出：完整有声剧音频
        ▼
┌─────────────────────────────────────────────┐
│            Phase 5: 质量闭环                 │
│  ┌──────────┐  ┌──────────┐                 │
│  │ 自动质检  │→│ 重写/修正 │                 │
│  └──────────┘  └──────────┘                 │
└─────────────────────────────────────────────┘
        │
        ▼
    最终输出（音频文件 + 元数据）
```

### 2. TTS 参数映射

| 情感 | 语速 | 音调 | 停顿 | 音量 |
|------|------|------|------|------|
| calm | 0.85 | 0 | +200ms | 0.8 |
| excited | 1.2 | +2 | +50ms | 1.0 |
| sad | 0.75 | -1 | +300ms | 0.6 |
| angry | 1.1 | +3 | +100ms | 1.1 |
| solemn | 0.9 | -1 | +250ms | 0.9 |
| fearful | 1.15 | +1 | +150ms | 0.5 |

### 3. 音频编排规则

```
[段落开始]
  ├── 静音 500ms
  ├── BGM 渐入（2秒）
  ├── 旁白/对话音频
  │     ├── 段落间静音 300ms
  │     └── 角色切换静音 500ms
  ├── 场景切换音效（可选）
  └── BGM 渐出（2秒）
[段落结束]
```

## 关键设计决策

### 1. 为什么用 JSON 而不是直接流式生成？

- **可检查**：中间产物可人工审核
- **可重试**：单段失败不影响整体
- **可并行**：多段音频可同时合成
- **可追溯**：每段都有元数据，方便定位问题

### 2. 声纹一致性方案

```
角色创建 → 提取声纹特征 → 存入向量库
    ↓
每章合成前 → 查询向量库 → 确保同一角色使用相同音色参数
    ↓
合成后 → 声纹比对 → 不一致则告警/重试
```

### 3. 情感控制方案

```
文本情感分析（MiMo）
    ↓
情感标签 + 强度值
    ↓
TTS参数映射表
    ↓
动态调整：语速、音调、停顿、音量
    ↓
（可选）后处理增强：混响、均衡器
```
